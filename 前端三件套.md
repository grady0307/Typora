### 布局

- 网页布局第一准则：多个块级元素纵向排列找标准流，多个块级元素横向排列找浮动。
- 网页布局第二准则：先设置盒子的大小，再设置盒子的位置

#### Position

- static 静态定位
  - 标准流，无边偏移
- relative 相对定位
  - 相对自己原来的位置
  - 保留原来的位置，不脱标 
- absolute 绝对定位 
  - 如果没有祖先元素或祖先元素无定位（标准流或浮动），则以浏览器为准定位
  - 父亲有相对/绝对/固定定位以父亲为准
  - 脱标不占位置
- fixed 固定定位
  - 以浏览器可视窗口作为参照点
  - 和父元素无关
  - 不随滚动条滚动
  - 不占有原先位置，脱标
  - ![image-20221001212150729](C:\Users\良小辰\AppData\Roaming\Typora\typora-user-images\image-20221001212150729.png)
- sticky 粘性定位
  - 以浏览器可视窗口为参照点
  - 占有原先的位置
  - 必须添加top,left,right,bottom其中一个才生效
- ==子绝父相==
- 叠放次序z-index
  - z-index:1;
  - 数字越大越靠上
  - 默认后来者居上
  - 只有定位的盒子才有z-index 
- ![image-20221001214144205](C:\Users\良小辰\AppData\Roaming\Typora\typora-user-images\image-20221001214144205.png)
- 定位和浮动类似，脱离标准流之后行内可以设置宽高，块级元素默认为内容大小（行内块元素的特征
- 脱标不触发外边距合并问题
- 浮动不会压住标准流盒子里面的文字（图片，但是定位会

- 一般来说一个盒子浮动了，它的兄弟元素也要浮动

- 浮动只会压住自己后面的标准流，不会压住自己前面的标准流

- 标准流的父盒子可以不指定高度而是让子盒子撑开父亲

- 清除浮动

  - 为什么需要

  - ![image-20221001162343527](C:\Users\良小辰\AppData\Roaming\Typora\typora-user-images\image-20221001162343527.png)

  - 本质：清除浮动元素所占有的影响

  - clear:both;

  - 方法

  - ![image-20221001162641056](C:\Users\良小辰\AppData\Roaming\Typora\typora-user-images\image-20221001162641056.png)

    1. 在最后一个浮动元素末尾添加一个空标签` <div style=" clear:both:></div>`要求必须是块级元素

    2.  父级添加overflow:hidden/auto/scroll;

    3. ~~~css
       //给父元素添加
       .clearfix:after{
       	content:"";
       	display:block;
       	height:0;
       	clear:both;
       	visibility:hidden;
       }
       .clearfix{
        /*IE6,7专有*/
           *zoom : 1;
       }
       <div class="clearfix"></div>
       ~~~

    4. 双伪元素

       ~~~css
       .clearfix:before,
       .clearfix:after
       ~~~
